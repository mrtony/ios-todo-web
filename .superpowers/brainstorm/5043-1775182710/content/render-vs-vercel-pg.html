方案對比

方案 A：Render 全包

架構：
Express API + Vite 靜態檔 → Render Web Service
PostgreSQL → Neon 免費方案

免費嗎？ ✅ 是

  • Render Web Service：免費（閒置 15 分鐘休眠）
  • Neon PostgreSQL：免費 0.5 GB

✅ 優點

  • 一個服務搞定，部署簡單
  • GitHub 連結自動部署
  • 標準 Node.js 環境，除錯方便
  • 可用 ORM (Prisma/Drizzle) 連 Neon

❌ 缺點

  • 休眠後首次請求慢 (~30s)
  • 兩個平台要管（Render + Neon）
方案 D：Vercel + GitHub Pages

架構：
前端靜態檔 → GitHub Pages
API → Vercel Serverless Functions
PostgreSQL → Neon 或 Vercel Postgres

免費嗎？ ✅ 是

  • GitHub Pages：免費
  • Vercel Functions：免費（每月 100GB）
  • Neon / Vercel Postgres：免費 0.5 GB

✅ 優點

  • 前端不休眠，載入速度快
  • Serverless 自動擴展，不用管伺服器
  • GitHub Pages 部署真正在 GitHub 上
  • Vercel + Neon 整合成熟

❌ 缺點

  • 需改寫 Express → Serverless 函數
  • 三個平台要管（GitHub + Vercel + Neon）
  • CORS 跨域設定
  • 除錯比傳統 Node.js 困難

我的推薦

兩個方案都是 完全免費 的。核心差異在於：

  • 方案 A — 開發體驗更好，傳統 Express 寫法，本地和線上環境一致，適合快速迭代
  • 方案 D — 前端載入更快（GitHub Pages 不休眠），但需要適應 Serverless 寫法

如果你的目標是快速把功能做出來，推薦 方案 A。
如果你很在意前端在 GitHub 上 + 不休眠，推薦 方案 D。
